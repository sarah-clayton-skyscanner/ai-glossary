🤖 AI Glossary

A simple guide to common words and phrases in AI and technology

@  the "at" symbol

When you type @ while chatting with Claude, it brings up a list of files in your project so you can pick one to include. It's like tagging a friend in a social media post — except instead of a person, you're pointing Claude to a specific file and saying "look at this one." 📎

See also: Skills

🤝  Agents (or Subagents)

These are helper versions of Claude, each trained for a specific job. Think of them like staff at a restaurant — there's a chef, a waiter, and a host. When Claude needs to do something specialist (like deep research), it hands that job to the right "staff member" so things get done better and faster.

See also: Tools, MCP

🚀  Agentic

Describes AI that doesn't just answer questions but takes a sequence of actions on its own to complete a goal. Instead of you doing every step, an agentic AI plans, decides, and acts — checking its own work along the way. It's the difference between asking someone a question and delegating a whole project to them. Claude Code is agentic: you give it a task, and it figures out the steps itself.

See also: Agents, Tools

🔗  API (Application Programming Interface)

A way for two pieces of software to talk to each other. You never see it — it works invisibly in the background. Think of it like a waiter in a restaurant: you don't go into the kitchen yourself, you tell the waiter what you want, and they bring back the result. For example, when an app shows you the weather, it's using an API to ask a weather service for the data.

See also: MCP

🏗️  Brownfield

A project that already exists — it has old code, history, and quirks built up over time. Working in a brownfield project means you have to be careful not to break things that are already there. Think of it like renovating an old house: you can't just knock down walls without checking what's behind them first.

See also: Greenfield, Repo

📋  CLAUDE.md

A plain text file where you write down the rules and preferences you want Claude to follow in your project. It's like leaving a note for a house-sitter: "Don't feed the cat after 6pm, the wifi password is X, always lock the back door." Claude reads this at the start of every session.

See also: Hooks, Permissions

🧠  Context Window

This is Claude's short-term memory for your conversation. It holds everything said and done so far in a session. When it gets too full, Claude automatically summarises the older parts to make room — like clearing the whiteboard once it's covered in notes, but keeping a summary of what was on it.

See also: Token, RAG

🧑‍✈️  Copilot

An AI assistant built directly into a tool you're already using, so it can help you without switching context. Microsoft Copilot sits inside Word, Excel, and Teams. GitHub Copilot helps developers write code inside their editor. The idea is that the AI rides alongside you, ready to help at any moment — rather than being a separate thing you have to go and visit.

See also: Agents, MCP

⌨️  Ctrl+G

A keyboard shortcut that opens your message in a proper text editor (like Notepad or VS Code) instead of typing in the small terminal window. It's useful when you want to write a long, detailed prompt and need more space to work. Once you save and close the editor, your text comes straight back into Claude Code, ready to send.

See also: TUI

🎭  Deepfake

AI-generated video or audio that makes a real person appear to say or do something they never did. The technology has become convincingly realistic, which makes deepfakes a growing concern for misinformation, fraud, and impersonation. If you see a video of a public figure saying something shocking, it's always worth checking whether it might be AI-generated before sharing it.

See also: Hallucination, Training Data

📝  Eval (Evaluation)

A test you run to see how well an AI is performing. Just like a teacher gives students an exam to check what they've learned, an eval gives Claude a set of questions or tasks to see how accurate and reliable its answers are. Teams use evals to spot weaknesses and improve the AI over time.

See also: Hallucination

🔧  Fine-tuning

Taking a general-purpose AI and training it further on specific material — like company documents, tone of voice guidelines, or industry data — so it becomes more expert in that area. It's like hiring a brilliant generalist and then sending them on a specialist training course. The core skills stay the same; they just get sharper in one particular area.

See also: Training Data, LLM

🌱  Greenfield

A brand new project built from scratch — no old code, no baggage, a completely blank slate. It's called "greenfield" because you're building on fresh, empty land. It's exciting because you can do things however you like, but it also means making a lot of decisions from the very beginning.

See also: Brownfield, Repo

🛡️  Guardrails

Rules and limits built into an AI to stop it from doing harmful, embarrassing, or unethical things. They're the invisible fences that keep AI behaviour within acceptable boundaries — preventing it from giving dangerous advice, generating offensive content, or going off-script in ways that could cause problems. Most AI tools you'll encounter at work have guardrails set by whoever deployed them.

See also: Permissions, Hallucination

🖥️  GUI — Graphical User Interface

The visual, clickable world most of us are used to — windows, buttons, menus, icons. When you use an app on your phone or click around on your laptop, you're using a GUI. It's designed to be easy and intuitive, so you don't need to know any technical commands. Pronounced "gooey."

See also: TUI

👻  Hallucination

When an AI confidently says something that is completely made up or wrong. It's not lying on purpose — it genuinely doesn't know it's wrong. Think of it like someone who misremembers a fact but says it with total confidence. This is why it's always worth double-checking important information that an AI gives you.

See also: Eval, Guardrails

🪝  Hooks

These are automatic actions that happen at set moments — like a reminder that goes off every time you save a document. For example, you could set a hook so that every time Claude edits a file, it automatically tidies up the formatting. You set it once, and it just runs by itself.

See also: Skills, CLAUDE.md

🗂️  Indexing Phase

The preparation stage where documents are read, broken up, and filed away so an AI can search through them quickly later. Think of it like a librarian organising a new delivery of books — they read the titles, sort them by topic, and put them on the right shelves before anyone comes to browse. This happens once upfront so that searches later are fast and accurate.

See also: RAG, Vector

🔓  Jailbreak

An attempt to trick an AI into bypassing its safety guardrails — getting it to say or do things it's been designed to refuse. Usually done by framing requests in clever or indirect ways, like telling it to pretend it's a different AI with no rules. It's a bit like trying to convince a bouncer you're on the guest list when you're not. AI companies are constantly working to close these loopholes.

See also: Guardrails, Permissions, Prompt Engineering

🧩  LLM (Large Language Model)

The type of AI that powers tools like Claude, ChatGPT, and Gemini. It's been trained on enormous amounts of text — books, websites, articles, code — and learned to understand and generate human language from all of it. The "large" part refers to just how much it has read and how complex it is under the hood. When people talk about "AI" in a work context, they're usually talking about an LLM.

See also: Training Data, Token, Fine-tuning

🔌  MCP (Model Context Protocol)

Think of this as a plug socket. It lets Claude connect to other apps and services — like plugging a lamp into the wall to get power. For example, Claude can plug into GitHub or a database to read and use information from there.

See also: API, Agents

🖼️  Multimodal

An AI that can understand and work with more than just text — like images, audio, PDFs, or spreadsheets. For example, you can show a multimodal AI a photo and ask "what's in this picture?" or hand it a document and ask it to summarise. Most modern AI assistants are now multimodal, which is what makes them useful for real-world work tasks rather than just chat.

See also: LLM, RAG

🔒  Permissions

These are the rules about what Claude is and isn't allowed to do without asking you first. It's like parental controls — you can say "always ask me before deleting anything" or "you can edit files freely." This keeps you in control.

See also: CLAUDE.md, Hooks

✏️  Prompt

The message or question you type to an AI — your side of the conversation. It's the instruction, request, or question that tells the AI what you need. A vague prompt tends to get a vague answer; a clear, specific prompt tends to get a much more useful one. Everything else in AI builds on top of this one simple idea.

See also: Prompt Engineering, Context Window

🎯  Prompt Engineering

The skill of writing prompts that get better results from an AI. It sounds fancier than it is — it's basically learning that "summarise this in 3 bullet points for a non-technical audience" gets a more useful answer than just "summarise this." The more specific and clear you are about what you want, who it's for, and what format you'd like, the better. No coding required.

See also: Prompt, ultrathink

📚  RAG (Retrieval-Augmented Generation)

A way of giving an AI access to a library of documents so it can look things up before answering. Instead of relying only on what it was trained on, the AI can fetch relevant information in real time — like being allowed to check your notes during an exam. This makes answers much more accurate and up to date.

See also: Indexing Phase, Vector, Context Window

📁  Repo (Repository)

A folder where all the files for a project live — including the full history of every change ever made to them. Think of it like a Google Doc that never forgets anything: you can see who changed what, when, and why. Repos are usually stored on a platform like GitHub so the whole team can access them.

See also: Worktree, Greenfield, Brownfield

📦  Sandboxing

A safety technique where code or AI actions run in an isolated environment, cut off from the rest of your system, so that if something goes wrong it can't cause damage outside that contained space. Think of it like testing a new recipe in a tiny batch before making it for a hundred people — you get the learning without the risk. It's one of the ways AI tools are made safer to use on real systems.

See also: Permissions, Guardrails

🕵️  Shadow AI

When employees use AI tools that haven't been approved or set up by their company's IT or security team. It often happens with good intentions — people just want to get things done faster — but it can create risks around data privacy, security, and compliance. If you're pasting work documents into a free AI tool, it's worth checking whether your company has an approved alternative.

See also: Guardrails, Permissions

⚡  Skills (also called Slash Commands)

These are shortcuts you can type to give Claude a special set of instructions. You type a / followed by a name, like /review. It's like pressing a button on a microwave that says "Popcorn" — instead of setting the time yourself, it does it all for you automatically.

See also: Hooks, @

🪃  Slingshot

Using AI to get a huge running start on something that would normally take a long time. Instead of building from zero, you use Claude to rapidly produce a first version — a draft, a plan, a block of code — and then refine it from there. Like a slingshot launching a stone much further than you could throw it by hand.

See also: Vibecoding

🪤  Slopsquatting

A security risk that combines AI hallucination with a classic hacker trick. When an AI invents the name of a software package that doesn't actually exist, an attacker can register that fake name as a real package — and fill it with malicious code. A developer who follows the AI's advice and installs it accidentally installs the malicious version instead. It's a new type of trap made possible by AI's tendency to make things up confidently.

See also: Hallucination, Guardrails

🌡️  Temperature

A setting that controls how creative or unpredictable an AI's responses are. Low temperature means careful, consistent, predictable answers. High temperature means more creative, varied, and sometimes surprising ones. It's like the difference between asking someone to proofread a legal contract (low temperature) versus brainstorm wild ideas for a campaign (high temperature). Most AI tools set this behind the scenes, but it's useful to know it exists.

See also: LLM, Prompt

🪙  Token

The unit an AI uses to process text — roughly three-quarters of a word. When you hear about "token limits," it means there's a cap on how much text can be handled in one go. Think of tokens like the credits on a pay-as-you-go phone: every word you type (and every word the AI writes back) uses a few. This is why very long conversations can slow down or hit limits.

See also: Context Window, LLM

🛠️  Tools

These are Claude's hands. Without tools, Claude can only talk. With tools, it can actually do things — like read a file, edit code, run a command, or search the web. Each tool is a specific action Claude is allowed to take.

See also: Agents, MCP

📖  Training Data

The enormous collection of text (and other content) that an AI was taught on before you ever met it — books, websites, articles, code, and more. It's effectively the AI's entire education. The quality and range of the training data shapes what the AI knows, how it communicates, and where its blind spots might be. It also explains why AI knowledge has a cut-off date: it only knows what it read up to a certain point.

See also: LLM, Fine-tuning, Hallucination

⬛  TUI — Terminal User Interface

A text-only interface where everything is typed rather than clicked. No buttons, no icons — just words and characters on a plain screen. Claude Code itself is a TUI. It might look intimidating at first, but it's actually very powerful once you get used to it. Think of it as the difference between texting someone instructions versus using a touchscreen app.

See also: GUI

💭  ultrathink

A special word you can type in your message to make Claude slow down and think much more deeply before answering. There's a whole dial of these — think, think hard, think harder, and ultrathink — each one asking Claude to put in more mental effort than the last. Use it when a problem is complicated and you want Claude's most careful, thorough answer rather than a quick one.

See also: Prompt Engineering, Eval

📐  Vector

A list of numbers that represents the meaning of a word, sentence, or document in a way a computer can understand. Words with similar meanings end up with similar numbers, so the computer can find related things even if they don't use the exact same words. It's like giving every idea in your library a unique postcode — things that are close in meaning get postcodes near each other, making them easy to find together.

See also: RAG, Indexing Phase

✨  Vibecoding

Writing software by describing what you want in plain English and letting AI do the actual coding for you — guided by feel and instinct rather than deep technical knowledge. You focus on what you want to build, and the AI figures out how to build it. It's made coding accessible to people who wouldn't traditionally call themselves developers.

See also: Slingshot

🌿  Worktree

A separate copy of your project that Claude can work on without touching your main version. Imagine photocopying a document before letting someone mark it up in red pen — the original stays safe.

See also: Repo
